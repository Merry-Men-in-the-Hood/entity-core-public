# Postmortems

When Entity experiences a SEV-0/1/2 incident, a postmortem is published
here within 72 hours, organized by date.

## Status

No incidents yet.

When they occur, and they will, they will be recorded here in full.
Wrong calls. Missed threats. False alarms. Drift events. Halts. The
ledger does not get curated. Mistakes are how Entity improves, but only
if it does not hide them.

## Format

Each postmortem follows the structure defined in
[`../governance/postmortem_template.md`](../governance/postmortem_template.md):

- Incident summary (ID, severity, detection/resolution times)
- What happened (factual description)
- Impact (users, funds, alerts, posts, trust)
- Timeline (signal to resolution)
- Root cause + contributing factors
- What worked / what did not
- Action items
- Public apology if applicable
- Lessons learned

## Naming

`ENTITY-YYYY-MM-DD-NN.md` where NN is a zero-padded sequence within the day.

Example: `ENTITY-2026-05-09-01.md`

## Severity definitions

See [`../governance/postmortem_template.md`](../governance/postmortem_template.md)
for full definitions of SEV-0 (Prime Directive violation),
SEV-1 (significant defect), and SEV-2 (minor defect).
